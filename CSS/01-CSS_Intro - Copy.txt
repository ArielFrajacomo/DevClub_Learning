    <h2>Lesson 004 - Fonts</h2>
    <!-- #region Lesson 004 - Fonts -->
    <p>We can import a web font from the HTML <code>&lt;head&gt;</code> using a <code>&lt;link&gt;</code> tag. 
       This is usually faster than using <code>@import</code> in the CSS file, because the browser discovers the font stylesheet earlier.</p>
    <p><span>&lt;link&gt;</span></p>
    <pre>
&lt;link rel="preconnect" href="https://fonts.googleapis.com"&gt;
&lt;link rel="preconnect" href="https://fonts.gstatic.com" crossorigin&gt;
&lt;link href="https://fonts.googleapis.com/css2?family=IBM+Plex+Sans:wght@300;400;500;600;700&amp;display=swap" rel="stylesheet"&gt;
    </pre>
    <p><span>@import</span></p>
     <pre>@import url('https://fonts.googleapis.com/css2?family=IBM+Plex+Sans:wght@300;400;500;600;700&display=swap');</pre>
    <p>Then, you can use the font-family property to apply the font to your elements:</p>
    <pre>  body {
        font-family: 'IBM Plex Sans', sans-serif;
    }</pre>
    
    <a href="https://fonts.google.com/" target="_blank"> Visit Google Fonts</a>
    <p><strong>Observation:</strong> <code>@import</code> was used in the CSS file to keep the HTML files simpler.</p>

    <h2>Lesson 005 - Margin, Padding, Border and Box Sizing</h2>
    <!-- #region Lesson 005 - Margin, Padding, Border and Box Sizing -->
    <p><strong>Margin</strong> is the space outside the border of an element, while <strong>padding</strong> is the space between the content of an element and its border. 
       The <strong>border</strong> is the line that surrounds the padding and content of an element.</p>
    <p>By default, the width and height of an element only include the content, and do not include padding and border. 
       This can lead to unexpected layout issues when you add padding and borders to an element.
         To fix this, you can use the <code>box-sizing: border-box;</code> property, which makes the width and height of an element include the padding and border.</p> 
    <pre>  .box {
        width: 200px;
        padding: 20px;
        border: 5px solid black;
        box-sizing: border-box;
    }</pre>
    <p>With <code>box-sizing: border-box;</code>, the total width of the element will still be 200px, and the content area will shrink to accommodate the padding and border.</p>
    <p></p>As a rule of thumb, it's generally a good idea to set <code>box-sizing: border-box;</code> on all elements to make layout easier and more predictable.</p>
    <pre>  * {
        box-sizing: border-box;
    }</pre>
    <!-- #endregion Lesson 005 - Margin, Padding, Border and Box Sizing -->
    <h2>Lesson 006 - Display</h2>
    <!-- #region Lesson 006 - Display -->
    <p>The <code>display</code> property in CSS is used to specify how an element should be displayed on the page. </p>
    <p><strong>Block</strong> elements take up the full width of their container and start on a new line. Examples include <code>&lt;div&gt;</code>, <code>&lt;h1&gt;</code>, and <code>&lt;p&gt;</code>.</p>
    <p><strong>Inline</strong> elements only take up as much width as necessary and do not start on a new line. Examples include <code>&lt;span&gt;</code>, <code>&lt;a&gt;</code>, and <code>&lt;strong&gt;</code>.</p>
    <p><strong>Inline-block</strong> elements are similar to inline elements, but they can have a width and height. Examples include <code>&lt;img&gt;</code> and <code>&lt;button&gt;</code>.</p>
    <p><strong>Flex</strong> and <strong>grid</strong> are layout models that allow you to create complex and responsive layouts. Flexbox is a one-dimensional layout model that arranges items in a row or column, while Grid is a two-dimensional layout model that allows you to create grid-based layouts.</p>
    <p class="card"><strong>Obs:</strong> Flexbox and Grid will be covered in more detail in future lessons.</p>
